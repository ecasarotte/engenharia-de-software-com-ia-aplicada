## Como transformar objetos JavaScript em tensores

TensorFlow.js **não consegue trabalhar diretamente com objetos JavaScript normais** (como `{ nome: "Carlos", idade: 40, cor: "verde", localizacao: "Curitiba" }`).
Internamente, ele precisa de um **tensor** – uma estrutura de dados que contém apenas números.
Abaixo está um guia passo a passo que usa o exemplo que vimos juntos: o conjunto de dados `pessoas` e o array auxiliar `categorias`.

---

### 1. Entenda o que precisamos transformar

```js
const pessoas = [
  { nome: "Erick", idade: 30, cor: "azul",   localizacao: "São Paulo" },
  { nome: "Ana",   idade: 25, cor: "vermelho", localizacao: "Rio" },
  { nome: "Carlos", idade: 40, cor: "verde",  localizacao: "Curitiba" }
];
```

Cada objeto tem:

| Propriedade | Tipo      | Como armazenar como número |
|----------|-----------|------------------------|
| `idade`  | numérico  | já é um número          |
| `cor`    | categórico| precisamos de um vetor unário (azul → [1,0,0], vermelho → [0,1,0], verde → [0,0,1]) |
| `localizacao` | categórico | vetor unário (São Paulo → [1,0], Rio → [0,1], Curitiba → [0,0]) |

---

### 2. Mapeie valores categóricos para índices numéricos

Primeiro, criamos um pequeno mapa para cada coluna categórica.
No contexto, já temos um array `categorias` que nos diz quais valores são importantes:

```js
const categorias = [
  'Erick', 'premium',
  'Ana',   'medium',
  'Carlos', 'basic'
];
```

Para **cor** e **localizacao** podemos construir um simples objeto de mapeamento:

```js
// extraímos as cores únicas da amostra (azul, vermelho, verde)
const cores = ['azul', 'vermelho', 'verde'];

// cria um mapa: cor → índice na lista de cores
const corParaIdx = {};
cores.forEach((c, i) => { corParaIdx[c] = i; });

// o mesmo para localizações (São Paulo, Rio, Curitiba)
const locs = ['São Paulo', 'Rio', 'Curitiba'];
const locParaIdx = {};
locs.forEach((l, i) => { locParaIdx[l] = i; });
```

---

### 3. Converta cada objeto em um vetor numérico

Agora percorremos o array `pessoas`, pegamos a idade, codificamos `cor` e `localizacao` como vetores unários e os juntamos.

```js
// helper para criar vetor unário
function oneHot(value, mapping) {
  const arr = new Array(Object.keys(mapping).length).fill(0);
  if (mapping.hasOwnProperty(value)) {
    arr[mapping[value]] = 1;
  }
  return arr;
}

// constrói a matriz numérica bruta
const dadosNumerosos = pessoas.map(p => {
  const vetorCor = oneHot(p.cor, corParaIdx);
  const vetorLoc  = oneHot(p.localizacao, locParaIdx);
  // juntamos idade + cor (3) + localizacao (3) = 7 números por pessoa
  return [p.idade, ...vetorCor, ...vetorLoc];
});

console.log(dadosNumerosos);
// → [
//   [30, 1, 0, 0, 1, 0, 0],
//   [25, 0, 1, 0, 0, 1, 0],
//   [40, 0, 0, 1, 0, 0, 1]
// ]
```

*O que está acontecendo?*
- A idade permanece como está.
- `cor` é transformada em um vetor de 3 elementos (`[1,0,0]` para azul, etc.).
- `localizacao` vira um vetor de 3 elementos (`[1,0]` para São Paulo, etc.).

---

### 4. Crie um tensor do TensorFlow.js

Agora que temos uma matriz de números puros, podemos passar para o TensorFlow.js.

```js
import * as tf from '@tensorflow/tfjs';

// `dadosNumerosos` é um array de 3 linhas × 7 colunas
const tensor = tf.tensor2d(dadosNumerosos);

// inspecione o shape – [3, 7]
console.log(tensor.shape);   // [3, 7]
console.log(tensor.dtype);   // "float32"
```

`tf.tensor2d` é ideal para dados tabulares como este porque produz uma **matriz 2D** (amostras × características).

Se você só precisa de uma única linha (por exemplo, ao classificar um novo usuário), pode criar um tensor 1D e depois expandir as dimensões:

```js
// exemplo: novo usuário { idade: 28, cor: "azul", localizacao: "Rio" }
const novaIdade   = 28;
const novaCorIdx = corParaIdx['azul'];   // 0
const novaLocIdx = locParaIdx['Rio'];    // 1

// vetor unário para cor (3) + localização (3)
const novaCorOH   = oneHot('azul', corParaIdx);   // [1,0,0]
const novaLocOH   = oneHot('Rio', locParaIdx);    // [0,1,0]

// combine em um vetor de características
const features    = [novaIdade, ...novaCorOH, ...novaLocOH]; // [28,1,0,0,0,1,0]

// crie um tensor 2D com forma [1, 7] (uma amostra)
const novoTensor = tf.tensor2d([features], [1, 7]);
```

---

### 5. (Opcional) Libere os tensores quando terminar

```js
tensor.dispose();   // se você não precisar mais dos dados antigos
novoTensor.dispose(); // libera o tensor do novo usuário
```

---

## Resumo rápido

| Passo | O que fazer | Por que é importante |
|------|------------|----------------|
| **Extraia** valores numéricos dos objetos | `p.idade`, `p.cor`, `p.localizacao` | TensorFlow.js só entende números |
| **Codifique** características categóricas | vetor unário (one‑hot) usando um mapa de índice | Transforma texto em uma representação numérica |
| **Junte** tudo em uma matriz de números | `pessoas.map(...)` | Prepara os dados para `tf.tensor2d` |
| **Crie** o tensor | `tf.tensor2d(dadosNumerosos)` | Agora o modelo pode processar os dados |
| **Libere** quando terminar | `tensor.dispose()` | Evita vazamentos de memória no navegador |

---

### Analogia do mundo real

Pense em um objeto JavaScript como uma **foto em PNG** – rico em informações, mas não compreensível para um programa de baixo nível.
Um tensor é como um **arquivo de texto simples** que contém apenas os mesmos fatos, mas em números que um algoritmo de machine learning pode ler rapidamente.

Ao converter cada foto (objeto) em uma lista simples de números (tensor), você está essentially **exportando os dados para um formato que o TensorFlow.js entende**.
